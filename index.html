Setup

Pacotes do Sublime: EditorConfig, Sass Syntax Highlighter, JS Beautify.

É necessário ter instalado na máquina NodeJS, NPM e Gulp.
Para verificar se os mesmos já estão instalados: 

nodejs -v
npm -v
gulp -v

Caso contrário: 

sudo apt-get install nodejs-legacy npm -y
sudo npm i gulp -g

NodeJS + NPM

Usamos o Node como plataforma para rodar as tarefas de compilação dos assets do sistema.
Para gerenciar o pacotes necessários para rodar essas tarefas é utilizado NPM - Node Package Manager.

Esses pacotes são definidos no arquivo package.json. 

{
    "name": "landing-page",
    "version": "1.0.0",
    "dependencies": {
        "gulp": "~3.8.11",
        "gulp-concat": "^2.4.1",
        "gulp-sass": "~1.3.3",
        "gulp-uglify": "^1.0.1"
    }
}

Nesse caso estamos definindo as seguintes dependências:

  • gulp versão próxima a 3.8.11
  • gulp-concat versão maior que 2.4.1
  • gulp-sass versão próxima a 1.3.3
  • gulp-uglify versão maior que 1.0.1

Definidas as dependências do projeto, para instalá-las basta rodar: 

sudo npm i
O NPM lerá o arquivo e instalará localmente (projeto) as dependências no diretório node_modules. Para adicionar uma dependência: 
sudo npm i gulp-sourcemaps
Para remover: 
sudo npm uninstall gulp-sourcemaps

Gulp

O que é?

Gulp é um task runner.

O que faz?

Não minifica, concatena ou altera nenhum arquivo, o que ele faz é rodar um conjunto de ações (tarefas) em arquivos através de plugins/ módulos javascript na plataforma NodeJS.
Estas tarefas são definidas no arquivo gulpfile.js.

Primeiramente, deve-se importar os módulos para compilação dos arquivos - definidas no package.json 

var gulp: require('gulp'),
    concat: require('gulp-concat'),
    sass: require('gulp-sass'),
    minifyJs: require('gulp-uglify');

Após isso, é definida uma tarefa, nesse exemplo, a compilação de um arquivos Sass/ Compass. 

gulp.task('site-css', function () {
    return gulp.src('css/style.scss')
        .pipe(sass())
        .pipe(concat('style.min.css'))
        .pipe(gulp.dest('dist'));
});

E enfim executamos: 

gulp site-css

Monitorando arquivos

Para não ser necessário executar a tarefa site-css toda vez que alterar o arquivo style.scss, podemos utilizar a feature watch. 

gulp.task('site-watch', function() {
    gulp.run('site-css');

    gulp.watch('css/style.scss', 'site-css');
});

Estamos informando ao Gulp que toda vez que o estilo do site for alterado, ele deve ser compilado novamente, executando: 

gulp site-watch

Sass + Compass

Sass

Linguagem de desenvolvimento que compila para CSS.
Guiada por identação.

Compass

Integrada ao Sass, possui as mesmas características que o mesmo, porém oferece sintaxe similar ao CSS, com {} e ;

Entre as características a primeira que se destaca é o aninhamento de seletores, para comparação. 

.container { font-size: 16px; }

.container p .tag { background-color: #606060; }

.container p .tag.new { background-color: #303030; }

.container p .tag > i { color: #fff; }
.container {
    font-size: 16px;

    p .tag {
        background-color: #606060;

        &.new { background-color: #303030; }

            > i { color: #fff; }
        }
    }

Mixins: 

@mixin big($color) {
    font-size: 26px;
    height: 100px;
    width: 300px;
    color: $color;
}

.button-large { @include big(#ffffff); }

.div-large { @include big(#606060); }

Variáveis: 

$main-color: #ffffff;

.alert { color: $main-color; }

.article { background-color: $main-color; }
